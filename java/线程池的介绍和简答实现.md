
为什么要使用线程池？
```
在面向对象的编程中创建对象和销毁对象非常的消耗性能也很费时间，在java中虚拟机创建对象之后虚拟还要跟踪对象，以便进行垃圾回收。为了解决这个问题人们就认为减少对象的创建和销毁的次数，并且运用现在已有的对象来解决当前的问题，这样池化技术就产生了。
```


池化技术是如何来提高服务器的性能的

```
讨论范围:接受客户的请求并且处理请求

多线程技术：
    主要主要解决多个线程的执行减少cpu的闲置，提高cpu的利用率，增加cpu的吞吐率

    多线程运用不当会造成的问题：
        (单线程)假设完成一个任务时间为T

        多线程
            T1 创建线程
                    T2线程执行任务的时间以及和其他线程同步的时间

                        T3销毁线程的时间

           T1+T2+T3>T

当我们在频繁的创建线程的时候这里突出弱点是T1,T3，而不是T2的并发性
当前我们渴望减少T1和T3所用的时间的比例，从而重整体上减少时间。





当我们每天50000以上请求数量的时候，我们比较利用线程池技术和不利用线程池技术服务器处理这些请求所产生的线程数

        线程池:
            线程的数量是固定的，所产生的线程总数不会超过线程池中的数目或者上线


        非线程
             有多少个请求就会创建多少个线程       
```
